form 태그 외부에 입력대그를 두어도 정보가 서버로 전달되지 않는다

속성
action: 서버로 정보를 보내면 이 정보를 받아서 처리할 페이지를 줘야 한다
        아무것도 안주면 자기페이지를 다시 호출한다.
method: 정보를 전달하는 방식을 지정할 수 있다.
        get 형식
            서버로 정보를 보낼때 비교적 간단한 정보만 보낸다.
            브라우저와 서버가 정보를 주고 받을때 header 정보와 body정보를 
            주고 받는데 get 방식은 header 정보만 주고 받는다.
            많은 양의 데이터를 주고 받으려면 body부분이 필요한데  get 방식은 body가 없다.
            get은 4096바이트 이내의 작고 보안이 필요없는 정보를 주고 받을때 사용한다.
            주고받을 때 사용한다. 보내는 정보가 모두 노출된다. 
            장점:url 복사하면 아까 그 위치로 이동가능
        
        post 형식
            header와 body 값들을 body에 감춰서 보내서 url에 정보가 노출되지 않는다.
            많은 양의 데이터, 파일 전송, 보안을 요하는 데이터를 post 방식으로 보낸다.
            특별히 지정안하면 get방식이 기본이다.
            1) www-url-encoded
            2) multipark : 파일전송시
            3) json 전송

enctype: 파일 전송시에만 필요하다


name : from태그및 그안에 속한 input 태그들만 가질 수 있다.
       from, select, input, textarea 태그들은 서버로 값을 전송 할 수 있는데
       값을 전송할때 name속성값=value속성값 의 형태로 전달된다.
       예) <input name="myvalue" value="Hello" 형태로 전달된다.
       get방식 전송시 한글이나 공백은 다른 문자로 치환되서 전달되기 때문에
       서버에서 받았을 때 값이 깨질수 있다.
       그래서 encoding 이라는 작업을 수행해서 보내야 한다.
       만일 보내야 할 겂이 여러가면 & (앤퍼센트 라고 읽는다)로 구분을 해서 보낸다.
       예) myvalue=Hello&b=4&age=56 이런식으로 보낸다. 

id : 모든 태그가 h1, p, input ...
     브라우저 상에서 접근 전환, 서버와 아무 관계없음
     객체들 접근해서 값읽고 쓰기 
     파일 하나에 무조건 한개만 존재해야 한다.
     여러개 쓰면? html은 상관 안함 javascript가 첫번째 것만 인식한다.

class : 위와 동일, 무조건 여러개로 알아듣는다. 배열형태로 접근한다.
